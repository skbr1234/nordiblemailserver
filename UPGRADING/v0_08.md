# Upgrading from `v0.7.3` to `v0.8.0`

Version `0.8.0` includes both performance and security enhancements that require your data to be migrated to a new database layout. Luckily version `0.7.3` includes a migration tool which should make this process much easier than previous upgrades. In addition to the new layout, you will have to change the systemd service file to use the `CAP_NET_BIND_SERVICE` capability.

## Preparation
- Upgrade to version `0.7.3` if you haven't already. If you are on a version previous to `0.7.0`, you will have to do a manual migration of your data using the Command-line Interface.
- Create a directory where your data will be exported to, for example `/opt/nordiblemailserver-mail/export`.

## Systemd service upgrade (Linux only)
- Stop the `v0.7.3` installation:
  ```bash
  $ sudo systemctl stop nordiblemailserver-mail
  ```
- Update your systemd file to include the `CAP_NET_BIND_SERVICE` capability. Open the file `/etc/systemd/system/nordiblemailserver-mail.service` in a text editor and add the following lines under the `[Service]` section:
  ```
  User=nordiblemailserver-mail
  Group=nordiblemailserver-mail
  AmbientCapabilities=CAP_NET_BIND_SERVICE
  ```
- Reload the daemon:
  ```bash
  $ systemctl daemon-reload
  ```
- Do not start the service yet.

## Data migration
- Stop NordibleMailServer and export your data:

  ```bash
  $ sudo systemctl stop nordiblemailserver-mail
  $ sudo /opt/nordiblemailserver-mail/bin/nordiblemailserver-mail --config /opt/nordiblemailserver-mail/etc/config.toml --export /opt/nordiblemailserver-mail/export
  $ sudo chown -R nordiblemailserver-mail:nordiblemailserver-mail /opt/nordiblemailserver-mail/export
  ```

  or, if you are using the Docker image:

  ```bash
  $ docker stop nordiblemailserver-mail
  $ docker run --rm -v <NORDIBLEMAILSERVER_DIR>:/opt/nordiblemailserver-mail -it nordiblemailserver-mail /opt/nordiblemailserver-mail/bin/nordiblemailserver-mail --config /opt/nordiblemailserver-mail/etc/config.toml --export /opt/nordiblemailserver-mail/export
  ```
- Backup your `v0.7.3` installation:
  - If you are using RocksDB or SQLite, simply rename the `data` directory to `data-backup`, for example:
    ```bash
    $ mv /opt/nordiblemailserver-mail/data /opt/nordiblemailserver-mail/data-backup
    $ mkdir /opt/nordiblemailserver-mail/data
    $ chown nordiblemailserver-mail:nordiblemailserver-mail /opt/nordiblemailserver-mail/data
    ```
  - If you are using PostgreSQL, rename the database and create a blank database with the same name, for example:
    ```sql
    ALTER DATABASE nordiblemailserver RENAME TO nordiblemailserver_old; 
    CREATE database nordiblemailserver;
    ```
  - If you are using MySQL, rename the database and create a blank database with the same name, for example:
    ```sql
    CREATE DATABASE nordiblemailserver_old;
    RENAME TABLE nordiblemailserver.b TO nordiblemailserver_old.b;
    RENAME TABLE nordiblemailserver.v TO nordiblemailserver_old.v;
    RENAME TABLE nordiblemailserver.l TO nordiblemailserver_old.l;
    RENAME TABLE nordiblemailserver.i TO nordiblemailserver_old.i;
    RENAME TABLE nordiblemailserver.t TO nordiblemailserver_old.t;
    RENAME TABLE nordiblemailserver.c TO nordiblemailserver_old.c;
    DROP DATABASE nordiblemailserver;
    CREATE database nordiblemailserver;
    ```
  - If you are using FoundationDB, backup your database and clean the entire key range.
- Download the `v0.8.0` mail-server for your platform from the [releases page](https://github.com/nordible/mail-server/releases/latest/) and replace the binary in `/opt/nordiblemailserver-mail/bin`. If you are using the Docker image, pull the latest image.
- Import your data:

  ```bash
  $ sudo -u nordiblemailserver-mail /opt/nordiblemailserver-mail/bin/nordiblemailserver-mail --config /opt/nordiblemailserver-mail/etc/config.toml --import /opt/nordiblemailserver-mail/export
  ```

  or, if you are using the Docker image:
  
  ```bash
  $ docker run --rm -v <NORDIBLEMAILSERVER_DIR>:/opt/nordiblemailserver-mail -it nordiblemailserver-mail /opt/nordiblemailserver-mail/bin/nordiblemailserver-mail --config /opt/nordiblemailserver-mail/etc/config.toml --import /opt/nordiblemailserver-mail/export
  ```
- Start the service:
  ```bash
  $ sudo systemctl start nordiblemailserver-mail
  ```

  Or, if you are using the Docker image:
  ```bash
  $ docker start nordiblemailserver-mail
  ```

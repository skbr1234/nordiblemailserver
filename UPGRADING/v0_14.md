# Upgrading from `v0.13.x` to `v0.14.x`

## Binary installation

- Stop NordibleMailServer in **every single node of your cluster**. If you are using the systemd service, you can do this with the following command:

  ```bash
  $ sudo systemctl stop nordiblemailserver
  ```

- Backup your data following your database system's instructions. For example, if you are using RocksDB or SQLite, you can simply copy the `data` directory to a backup location. If you are using PostgreSQL or MySQL, you can use the `pg_dump` or `mysqldump` commands to create a backup of your database. If your database does not support backups, you can use the [built-in migration utility](https://nordible.co/docs/management/migration) to export your data to a file. For example:

  ```bash
  $ sudo /opt/nordiblemailserver/bin/nordiblemailserver --config /opt/nordiblemailserver/etc/config.toml --export /opt/nordiblemailserver/export
  $ sudo chown -R nordiblemailserver:nordiblemailserver /opt/nordiblemailserver/export
  ```

- Download the latest binary for your platform from the [releases page](https://github.com/skbr1234/mailadmin.nordible.co/releases/latest/) and replace the binary in `/opt/nordiblemailserver/bin`. 

- Start the service. In a cluster, you can speed up the migration process by starting all nodes at once. 
  ```bash
  $ sudo systemctl start nordiblemailserver
  ```

- Upgrade the webadmin by clicking on `Manage` > `Maintenance` > `Update Webadmin`.

## Containerized

- Stop the NordibleMailServer container in **every single node of your cluster**. If you are using Docker, you can do this with the following command:

  ```bash
  $ docker stop nordiblemailserver
  ```

- Backup your data following your database system's instructions. For example, if you are using RocksDB or SQLite, you can simply copy the `data` directory to a backup location. If you are using PostgreSQL or MySQL, you can use the `pg_dump` or `mysqldump` commands to create a backup of your database. If your database does not support backups, you can use the `--export` command to export your data to a file. For example:

  ```bash
  $ docker run --rm -v <NORDIBLEMAILSERVER_DIR>:/opt/nordiblemailserver -it nordiblemailserver /usr/local/bin/nordiblemailserver --config /opt/nordiblemailserver/etc/config.toml --export /opt/nordiblemailserver/export
  ```

- Pull the latest image and restart the container:

  ```bash
  $ docker pull nordible/nordiblemailserver:latest
  $ docker start nordiblemailserver
  ```
  
- Upgrade the webadmin by clicking on `Manage` > `Maintenance` > `Update Webadmin`.
